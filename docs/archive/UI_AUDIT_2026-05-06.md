### 6 spec amendments (Bundle C deliverable)

These are emerging "better patterns" the audit surfaced. Each gets promoted to a documented primitive AND migrated platform-wide. Both `RESTRUCTURE_SPEC.md` and `AVERROW_UI_STANDARD.md` get updated.

1. **`StatCard` zero-state rule** — `value === 0` resolves accent to `neutral` regardless of prop. Kills red-on-zero anti-pattern (M2)
2. **`ContextBadge`** — promotes NEXUS/PIVOT/ACCELERATING/QUIET/WORSENING/IMPROVING tags from Provider cards. Reusable on threat actor / campaign / cluster surfaces
3. **`VerdictPill`** — promotes pipeline verdict (CLEAR/DRAINING/STEADY/GROWING/STALE/UPDATED/STABLE) from Metrics. Reusable on any monitor / health surface
4. **`PriorityBar`** — promotes takedown priority bar (70/100 + colored progress). Reusable on alerts, leads, scoring rows
5. **`StateMachineButtons`** — promotes incident-detail state row. Reusable on takedown state, alert state, future workflow surfaces
6. **`EmptyState` variants locked** — codify the green-bell + target-icon + map-fallback patterns. Variants: `success | empty-list | data-unavailable | configure-me`
