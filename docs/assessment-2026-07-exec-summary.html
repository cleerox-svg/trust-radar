Averrow Platform · Capabilities & Gaps Assessment

The capability is real. The story it tells about itself is the gap.

Date 2026-07-17 Method 9 parallel read-only code lenses Audience Exec · Engineering · Positioning Supersedes PLATFORM_ASSESSMENT_2026-06

Averrow is a broad, genuinely strong, remarkably low-cost Digital Risk Protection engine. Detection breadth, standing-gated takedown execution, cost discipline, and the agent mesh are real and largely production-grade. Three things hold it back: a positioning & terminology layer that misnames its own capabilities and leaks internal code names to customers; a headline differentiator — "identify who conducts attacks" — the code doesn't yet deliver; and a handful of live operational and security sharp edges that are cheap to fix.

R1

A live starvation gap is dropping ~67% of three scanners' runsP0 · live

CT monitor, lookalike scanner, and trademark scan run inline at the tail of the hourly tick after a 153s Analyst wait and complete only 8 of 24 runs. ct_monitor has zero agent_runs telemetry, so the stall watchdog can't even see it fail. Same fix pattern already applied to nine other agents.

S1

One shared internal secret can mint an admin-mutation tokenP0 · security

The mint-ui-preview-jwt endpoint plus a single shared AVERROW_INTERNAL_SECRET can issue a 4-hour admin JWT reaching ~58 admin endpoints. The diagnostics/agent-trigger secret is effectively admin-equivalent if it leaks. Two independent lenses converged here.

S2

The "auditor" read-only role is a convention, not a mechanismP0 · security

It sits at staff hierarchy level 3, so any bare requireStaff-gated mutation (brands, email-security scan, investigations) accepts an auditor token — despite the role being designed to write nothing.

Sentinel / ASTRA / Observer / Navigator / Blackbox / Pathfinder are the primary labels on the marketing site and in the public changelog. "Blackbox" is a phantom — no such agent exists — and is aviation framing. Three customer descriptions are factually wrong: Navigator (geo vs DNS), Pathfinder (a sales tool sold as customer protection), Blackbox↔Narrator.

03 Takedown standing model — founder decision, verified in code

Averrow has no legal standing to submit a takedown for a brand until that brand is a customer with an authorizing tenant. The verification found this principle is already ~90% enforced — with two net-new surfaces and one integrity fix remaining.

Enforced today
  • ✓The sole outbound dispatcher (Sparrow Phase G) hard-excludes brands with no owning org and requires a per-org signed MSA + entitlement + provider opt-in + an auto policy decision. No path can fire a takedown for a non-customer.
  • ✓The org_brands join table is a clean customer-vs-prospect marker; auto / semi-auto / off posture is stored per tenant on the signed scope.
Remaining work
  • ▹TK1 — integrity fix. An admin status-flip can stamp any takedown "submitted" with no standing check, misrepresenting an unsent/unauthorized action as sent.
  • ▹TK2 — analyst hand-submit (net-new). No ops path for an Averrow analyst to dispatch an authorized customer's edge case. The primitives to build it cleanly already exist, unused.
  • ▹TK3 — Ops "Takedowns" becomes two surfaces (net-new, mostly composition). Authorized execution plus a per-brand prospect/pitch view — "everything we'd action for you" — composed from ingredients that already exist (exposure scan, BrandDetail Risk tab, orgless drafts already in the admin queue).

04 The differentiator decision — locked

Re-anchor now, invest next

Stop implying actor-fingerprinting the code doesn't do; lead with what's genuinely defensible — then build the claim back.

Now — re-anchor

Shift customer copy onto the free no-signup scan, the transparent "SQL correlates, AI narrates" stance, edge-native price/cost, standing-gated takedown execution, and campaign clustering. Retire "42 agents" as the headline — Group-IB now markets the same idea.

Next — invest, in ROI order

Close the "WHO" claim: the newly-registered-domain signal (the data is already on the wire, discarded) → favicon / JA3-JARM lanes + connected-components clustering + real infra-movement pivots → page-content / visual phishing → detection breadth.
